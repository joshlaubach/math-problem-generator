═══════════════════════════════════════════════════════════════
FORBIDDEN PATTERNS — NEVER USE THESE
═══════════════════════════════════════════════════════════════

• stroke_dasharray — this is SVG/CSS, NOT a Manim parameter. Use DashedLine instead.
• stroke_dash_pattern — same issue. Use DashedLine.
• cone_height= on Arrow3D — the parameter was renamed to height= in Manim 0.19.
• get_text(font_size=N) on Brace — use put_at_tip() with a separately created Text/MathTex.
• self.wait(0) or self.wait(negative) — Manim will crash; minimum is 0.001.
• self.wait("any_string") outside of BEAT_N pattern — only BEAT_N strings are injected.
• scale_to_fit_width(10) on equations placed at y > 2.0 — this causes title overlap.
  Step content must be placed at UP * 1.8 or lower when a title is at the top edge.
• Placing any mobject above y = 3.2 (this overlaps the safe-zone title area).
• Placing any mobject outside x ∈ [-6.5, 6.5] or y ∈ [-3.5, 3.5].
• Using Unicode characters like ½, ¼, ✓, ✗ inside MathTex/Tex — use LaTeX commands
  (r"\frac{1}{2}", r"\checkmark", r"\times") instead.
• RightAngle with a zero-length line argument — guard against degenerate lines.

═══════════════════════════════════════════════════════════════
SAFE ZONE LAYOUT RULES
═══════════════════════════════════════════════════════════════

Title area:   y ∈ [2.8, 4.0]  — reserved for the title (to_edge(UP, buff=0.3))
Content area: y ∈ [-3.0, 2.5] — all animations and equations go here
Bottom text:  y ∈ [-3.8, -3.2] — reserved for question/summary text

When placing step equations during a lesson:
  • Use move_to(UP * 1.5) or lower for step content (font_size ≤ 32)
  • Use scale_to_fit_width(8.0) maximum (not 10 or 12)
  • For fractions, use \tfrac{}{} (inline fraction, shorter height) instead of \frac{}{}
    when the equation must sit near the top of the content area.
